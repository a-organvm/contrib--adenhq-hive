# Session 01 — 2026-03-21

## Scope

Design and implement Phase 1 of the Hive Design Versioning system (issue #6613).

## Decisions

1. **Approach B selected** — Cross-organ symbiote. The contribution spans ORGAN-I (theory), ORGAN-II (visualization), ORGAN-III (shipped code), ORGAN-IV (orchestration), ORGAN-V (narrative). Not a single-repo contribution.

2. **Event bus over closure modification** — `confirm_and_build()` in Hive's `queen_lifecycle_tools.py` is a closure-scoped local function. Rather than invasively modifying it, we add a `DESIGN_VERSION_SAVED` event type to Hive's existing event bus. The versioning store subscribes to this event. This is idiomatic Hive and less invasive for a first contribution.

3. **Flat CLI pattern** — Hive uses flat subcommands (`test-run`, `test-debug`), not nested sub-subcommands. We follow with `version-list`, `version-show` instead of `versions list`, `versions show`.

4. **Phased PR strategy** — Phase 1 (this session): schemas, store, basic CLI, tests. Phase 2 (follow-up): lifecycle hooks via event bus, advanced CLI (`version-diff`, `version-restore`, `version-star`), runner integration.

5. **Forward-only lifecycle transitions** — Mirrors ORGANVM Article VI. `DRAFT -> CANDIDATE -> VALIDATED -> PROMOTED -> ARCHIVED`. No back-transitions. If a promoted version needs to be superseded, a new version goes through the full lifecycle.

## Observations

- Hive already has a `feature/agent-versioning` branch upstream (862 files changed, stale internal branch). Our approach is architecturally distinct — governed lifecycle state machine vs. basic versioning.
- The `Checkpoint`/`CheckpointStore`/`CheckpointIndex` triad is the perfect structural template. Our `DesignVersion` triad mirrors it exactly.
- `StateSnapshot.verify()` from agentic-titan maps directly to `DesignVersion.verify()` — integrity checksums for versioned artifacts.
- Hive's `QueenPhaseState` (planning -> building -> staging -> running) is a state machine without forward-only enforcement. Our lifecycle model demonstrates what governance looks like.
